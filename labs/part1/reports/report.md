# Шаг 1 — Установка необходимого программного обеспечения

На первом этапе устанавливаем необходимые инструменты для работы: генератор статических сайтов Hugo и систему контроля версий Git.

Открываем терминал и выполняем команды:

# Устанавливаем Hugo
sudo apt install hugo

# Альтернативный способ установки (может быть более свежая версия)
sudo snap install hugo

# Проверяем установленную версию
hugo version

# Устанавливаем Git
sudo apt install git

После выполнения команд в системе появляются необходимые инструменты для создания и публикации сайта.

---

# Шаг 2 — Скачивание шаблона сайта

В качестве основы сайта используем шаблон Academic для Hugo. Скачиваем его из репозитория GitHub.

git clone https://github.com/wowchemy/starter-hugo-academic.git my-site
cd my-site

После клонирования репозитория инициализируем подмодули, которые содержат темы и дополнительные компоненты сайта.

git submodule update --init --recursive

В результате получаем готовую структуру сайта.

---

# Шаг 3 — Создание репозитория на GitHub

Переходим на сайт GitHub и создаём новый репозиторий.

Название репозитория задаём в формате:

логин.github.io

Например:

ivanov.github.io

Тип репозитория выбираем Public.

После создания репозитория подключаем его к локальному проекту.

cd my-site

# удаляем старый origin
git remote remove origin

# подключаем свой репозиторий
git remote add origin https://github.com/логин/логин.github.io.git

# добавляем файлы
git add .

# создаём первый коммит
git commit -m "Initial site setup"

# отправляем проект в репозиторий
git push -u origin main

В результате файлы сайта загружаются в репозиторий GitHub.

---

# Шаг 4 — Настройка адреса сайта

Далее настраиваем адрес будущего сайта. Для этого открываем файл конфигурации:

hugo.yaml

Находим строку:

baseURL: 'https://example.com'

Заменяем её на адрес сайта:

baseURL: 'https://логин.github.io/'

После изменения конфигурации сохраняем файл и отправляем изменения в репозиторий.

git add .
git commit -m "Set base URL"
git push

---

# Шаг 5 — Настройка GitHub Pages и публикация сайта

Далее переходим в настройки созданного репозитория.

Открываем раздел:

Settings → Pages

В параметре Source выбираем вариант:

GitHub Actions

GitHub автоматически предлагает workflow для сборки сайта на Hugo. Подтверждаем предложенный workflow, после чего начинается автоматическая сборка и публикация сайта.

---

# Шаг 6 — Альтернативный способ публикации

Сайт также можно опубликовать вручную через ветку gh-pages.

Для этого собираем сайт:

hugo

После сборки папка public содержит готовый сайт.

Загружаем её в ветку gh-pages:

git subtree push --prefix public origin gh-pages

После этого в настройках репозитория в разделе Pages выбираем источник публикации:

Branch: gh-pages

---

# Шаг 7 — Проверка результата

После завершения сборки переходим в настройки репозитория:

Settings → Pages

В этом разделе отображается ссылка на опубликованный сайт.

Сайт становится доступен по адресу:

https://логин.github.io/

---

# Итог

В ходе выполнения работы создаём статический сайт с использованием генератора Hugo, подключаем его к репозиторию GitHub и публикуем с помощью сервиса GitHub Pages. В результате сайт успешно размещается в сети Интернет и становится доступным по постоянному адресу.
